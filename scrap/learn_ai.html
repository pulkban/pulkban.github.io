-----------LLM-----------
neural network that is trained to predict the next term of an input sequence
e.g. all that glitter is _ _ _ _ (gold)
LLM predicts next word based on input sequence

-----------tokenization (smallest) -----------
to process input sequence - break into tokens - smallest unit of meaning
- break into chunks e.g. "all" "that" "glitt" "ers" "eat" "ing"

-----------vectors -  -----------
trying to derive meaning from those small tokens by 'clustering'
llm in n-dimensional space - words of same meaning is clustered - and has a coordinate (0.00, 0.00)

-----------Attention Mechanism -----------
e.g. "tasty apple" "apple's revene" "apple of my eye" - apple has different meanings - by looking at nearby words - adds context to apple - 
you do attention operation: vector_Revenue + vector_Apple - 
then, you push into that direction

-----------how do u train LLM-----------
- 2017 - self supervised learning - the input stream is having some inherent structure with which you understand the pattern and predict the next word even in input is not there - e.g. 5...4...3...2..(blank) - u know it will be 1.
 
-----------Transformer-----------

output vectors -> attn block (might find more relationsihps) -> ffnn -> output
...

principle: get input -> get meaning -> manipulate again and again -> preduct

-----------fine tuning-----------
what kind of 'next token'
if financial - next token - in financial terms
if medical - next token - in medical terms
so, take the base model - make it go through series of similar questions
fine tuning process - penalizes undesirable responses - who is next us pres? "even i wanted to know"
e.g. base model llama + company fine tunes to a context => relevant responses
 
-----------few shot prompting-----------
instead of plain query to LLM - pad it with examples during the input e.g. where is my order (E.g. it is in transit and eta=11-Jun) - now, responses are quite good
called "example prompting"
