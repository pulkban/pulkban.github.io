Reverse the string -----------
public class Main {
    public static void main(String[] args) {
        String str = "hello world";
        char[] chars = str.toCharArray();
        int n = chars.length;

        for (int i = 0; i < n / 2; i++) {
            char temp = chars[i];
            chars[i] = chars[n - 1 - i];
            chars[n - 1 - i] = temp;
        }

        String reversed = new String(chars);
        System.out.println("Original: " + str);
        System.out.println("Reversed: " + reversed);
    }
}

Output:
Original: hello world
Reversed: dlrow olleh
        

Find 3rd largest number using streams -----------
import java.util.Arrays;
import java.util.Comparator;

public class Main {
    public static void main(String[] args) {
        int[] arr = {5, 1, 9, 3, 7, 9};

        int thirdLargest = Arrays.stream(arr)
                               .boxed()
                               .sorted(Comparator.reverseOrder())
                               .distinct()
                               .skip(2)
                               .findFirst()
                               .orElseThrow();

        System.out.println("3rd: " + thirdLargest);
    }
}

Output: 3rd: 5

public class Main {

        public MyException(String message) {
            super(message);
        }
    }

    static void checkValue(int value) 
        throws MyException {
        if (value < 0) {
            throw new MyException(
                "Value cannot be negative");

            System.out.println(
                "Value is OK: " + value);
        }
    }

    public static void main(String[] args) {
        try {
            checkValue(10);
            checkValue(-5);
        } catch (MyException e) {
            System.out.println(
                "Caught: " + e.getMessage());
        }
    }
}

Output:

Caught: Value cannot be negative
        

Atomic Operations and Thread Safety -----------
import java.util.concurrent.*;
import java.util.concurrent.atomic.*;

public class Main {
    public static void main(String[] args) 
        throws InterruptedException {
        AtomicInteger counter = 
            new AtomicInteger(0);
        Runnable task = () -> 
            counter.incrementAndGet();
        
        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);
        t1.start(); 
        t2.start();
        t1.join(); 
        t2.join();
        
        System.out.println(
            "Counter: " + counter.get());

        ConcurrentHashMap map 
            = new ConcurrentHashMap<>();
        map.put("A", 1);
        map.merge("A", 1, Integer::sum);
        System.out.println("Map: " + map);
    }
}

Output:
Counter: 2
Map: {A=2}

EventListener Pattern -----------
Write an EventListener Code: Lets a method listen to events published in your Spring application. - Decouples the publisher (sends event) and listener (handles event) - Can be synchronous or asynchronous (@Async).

Purpose: Method listens to published events
Decouples publisher and listener

class MyEvent {
    private final String message;
    public MyEvent(String msg) {
        this.message = msg;
    }
    public String getMessage() {
        return message;
    }
}

class MyEventListener {
    public void handleEvent(MyEvent e) {
        System.out.println(
            "Received: " + e.getMessage());
    }
}

public class MainApp {
    public static void main(String[] args) {
        MyEvent event = 
            new MyEvent("Hello!");
        MyEventListener listener = 
            new MyEventListener();
        listener.handleEvent(event);
    }
}

Output:
Received: Hello!
        

Spring WebFlux Reactive -----------
Spring WebFlux → handles requests non-blocking using reactive streams (Mono / Flux); instead of blocking threads for IO, it registers callbacks and reuses few threads to serve many requests ⚡

Mono = 0 or 1 result
Flux = 0…N results

Non-blocking, uses callbacks, reuses threads

@RestController
public class UserController {
    @GetMapping("/user/{id}")
    public Mono getUser(
        @PathVariable String id) {
        return userService.findById(id);
    }
    
    @GetMapping("/users")
    public Flux getAllUsers() {
        return userService.findAll();
    }
}

Usage:
Mono mono = 
    Mono.just(user);

Flux flux = 
    Flux.fromIterable(users);

Prevents thread blocking
Scales to many concurrent requests
        

Retryable vs Manual Retry -----------
@Retryable: Spring handles automatically
Manual: You write try-catch loop

@Retryable Approach -----------
@Retryable(
    value = RuntimeException.class, 
    maxAttempts = 3, 
    backoff = @Backoff(1000))
public void callService() { 
    // may fail
}

Manual Retry -----------
for (int i = 0; i < 3; i++) {
    try {
        callService();
        break;
    } catch (RuntimeException e) {
        Thread.sleep(1000);
    }
}

Comparison:
@Retryable: Cleaner, less code
Manual: Full control, complex logic
        

Distributed Locking in Spring -----------
When: Multiple service instances must not 
execute critical section simultaneously

Examples: scheduled jobs, shared resource 
update, payment processing

Tools: Redis, Zookeeper, DB locks

@Service
public class JobService {
    private final RedissonClient redisson;
    
    public void runJob() {
        RLock lock = 
            redisson.getLock("jobLock");
        try {
            if (lock.tryLock(5, 10, 
                TimeUnit.SECONDS)) {
                // critical work here
            }
        } finally {
            if (lock.isHeldByCurrentThread()) 
                lock.unlock();
        }
    }
}
        

Anti-Corruption Layer (ACL) -----------
Protects service from legacy/external models
Purpose: Protect your service from legacy or external system models; prevent “leaking” unwanted complexity into your domain.
How: Introduce a translator layer that converts external models → internal models.
Benefit: Keeps your microservice clean, consistent, and decoupled.

Translates external models to internal models

// External DTO
class ExternalUser {
    public String fullName;
    public String phone;
}

// Internal domain
class User {
    private String name;
    private String contact;
}

// ACL Translator
class UserTranslator {
    public static User fromExternal(
        ExternalUser e) {
        User u = new User();
        u.setName(e.fullName);
        u.setContact(e.phone);
        return u;
    }
}

Usage:
ExternalUser ext = new ExternalUser();
ext.fullName = "Alice";
ext.phone = "12345";

User internal = 
    UserTranslator.fromExternal(ext);

Benefit: Keeps service clean, decoupled

String Comparison Output -----------
What will be printed?

String a = "hello";
String b = "hello";
String c = new String("hello");

System.out.println(a == b);     // true
System.out.println(a == c);     // false
System.out.println(a.equals(c)); // true

Explanation:
a == b: Both point to same pool reference
a == c: a is pool, c is heap (different)
a.equals(c): Compares content (both hello)

String literal → goes to pool, reused
new String() → creates separate heap object

Dynamic Filters: Optional Params -----------
Fetch employees by name, dept, salary range

REPOSITORY
import org.springframework.data.jpa.
    repository.JpaSpecificationExecutor;

public interface EmployeeRepository extends 
    JpaRepository, 
    JpaSpecificationExecutor {
}

SPECIFICATION
public class EmployeeSpec {
    public static Specification 
        filter(String name, String dept, 
        Double minS, Double maxS) {
        return (root, query, cb) -> {
            var p = cb.conjunction();
            if (name != null && 
                !name.isEmpty())
                p = cb.and(p,
                    cb.like(root.get("name"), 
                    "%" + name + "%"));
            if (dept != null && 
                !dept.isEmpty())
                p = cb.and(p,
                    cb.equal(root.get("dept"), 
                    dept));
            if (minS != null)
                p = cb.and(p,
                    cb.greaterThanOrEqualTo(
                    root.get("salary"), minS));
            return p;
        };
    }
}
